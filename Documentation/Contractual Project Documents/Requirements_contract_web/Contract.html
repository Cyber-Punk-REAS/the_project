COVID-19 Spread Simulator

March 24th, 2021

PROBLEM TO SOLVE

With our SUD we intend to provide the pertinent organizations with a useful tool to guide them and to give them valuable information in order to apply more efficient measures to reduce the spread of the COVID-19, minimizing externalities.

PROPOSED SOLUTION

For this, we propose to create a program to simulate the spread of COVID-19, where the most commonly used measures against the virus can be tested to see how they perform in different regions and populations.

BENEFITS

This project will allow us to observe the evolution of the virus, which will turn out to be genuinely useful to forestall some other potential areas of infection, take the necessary steps, and prevent the virus from spreading among the population.

THE CONTRACT

Commitments

  • The program should allow modifying a given generic region given the next parameters:
  • Number of areas.
  • Per area:
  • Population.
  • Size.
  • Number of businesses.
  • The restrictions that the program has to consider are:
  • Curfew (time of curfew is assumed to be appropriate).
  • Opening/closing businesses (businesses being segregated by essential and non-essential).
  • Time restrictions for the activities of businesses (segregated into time spans where businesses are allowed to open; i.e. 2h, 4h, 6h, 8h) If businesses closed, does not apply.
  • Mobility restrictions (closing down certain areas of the region).
  • The program should be able to analyze which of all the measures used was the most effective at reducing the spread of the virus.

Extensions

  • If time permits, the program should provide information about the situation as measures are being taken and as the simulation is performed.
  • We would like to make a graphical interface that shows a map with colored areas where it is shown how the covid is advancing or decreasing, where the different colors represent the percentage of the population that is infected.
  • If the project happens to be a website, we could make it responsive.
  • Also, we have thought to do another graphical interface but with a chart instead of a map.
  • For more convenience, another option could be to develop an app for mobiles.
  • Allow the client to modify the algorithm’s parameters for the spread of the virus.
  • Tourism.

Exclusions

  • Complex behavior of individuals is discarded due to the computational complexity and the research that it implies.
  • No specific regions will be used in the simulations. Only generic modifiable regions.
